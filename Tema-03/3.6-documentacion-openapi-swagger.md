> **Duración:** ~45 minutos  
> **Contexto:** Documentar automáticamente catalog-api  
> **Hilo conductor:** ShopMicro - APIs autodescriptivas

---

## ¿Qué es OpenAPI / Swagger?

**OpenAPI** (antes **Swagger**) es un estándar abierto para describir APIs REST de forma machine-readable.

### Beneficios

✅ **Documentación automática:** Se genera del código  
✅ **Testeo interactivo:** Swagger UI para probar endpoints sin Postman  
✅ **Generación de clientes:** Herramientas pueden generar SDKs  
✅ **Contrato API:** Punto único de verdad para consumers  
✅ **Monitoreo:** Herramientas como SonarQube la usan para análisis  

---

## Setup de Swagger en Spring Boot (springdoc-openapi)

En Spring Boot el estándar de facto es **springdoc-openapi**, que genera la
especificación OpenAPI 3 y sirve Swagger UI automáticamente.

### Instalación (`pom.xml`)

```xml
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.6.0</version>
</dependency>
```

Solo con añadir la dependencia ya dispones de:
- `http://localhost:8080/swagger-ui.html` → Swagger UI
- `http://localhost:8080/v3/api-docs` → especificación OpenAPI (JSON)

### Configuración: metadatos y seguridad

En lugar de configurar `AddSwaggerGen(...)` en `Program.cs`, en Spring definimos un bean
`OpenAPI` en una clase `@Configuration`:

```java
package com.shopmicro.catalog.config;

import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Contact;
import io.swagger.v3.oas.models.info.Info;
import io.swagger.v3.oas.models.info.License;
import io.swagger.v3.oas.models.security.SecurityRequirement;
import io.swagger.v3.oas.models.security.SecurityScheme;
import io.swagger.v3.oas.models.Components;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class OpenApiConfiguration {

    @Bean
    public OpenAPI catalogOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .version("v1")
                .title("catalog-api")
                .description("Microservicio de catálogo para ShopMicro")
                .contact(new Contact()
                    .name("ShopMicro Team")
                    .email("support@shopmicro.local")
                    .url("https://shopmicro.local"))
                .license(new License()
                    .name("MIT")
                    .url("https://opensource.org/licenses/MIT")))
            .addSecurityItem(new SecurityRequirement().addList("bearer-jwt"))
            .components(new Components()
                .addSecuritySchemes("bearer-jwt", new SecurityScheme()
                    .type(SecurityScheme.Type.HTTP)
                    .scheme("bearer")
                    .bearerFormat("JWT")
                    .description("JWT Authorization header using the Bearer scheme.")));
    }
}
```

### Ajustes en `application.yml` (opcional)

```yaml
springdoc:
  swagger-ui:
    path: /swagger-ui.html
    operations-sorter: method
  api-docs:
    path: /v3/api-docs
```

## Documentar endpoints con anotaciones

springdoc usa las anotaciones de **swagger-core** (`@Operation`, `@ApiResponse`, etc.),
equivalentes a los comentarios XML y a `[ProduceResponseType]` de .NET.

### Ejemplo: CreateProduct

```java
package com.shopmicro.catalog.features.products.create;

import an.awesome.pipelinr.Pipeline;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.media.Content;
import io.swagger.v3.oas.annotations.media.Schema;
import io.swagger.v3.oas.annotations.responses.ApiResponse;
import io.swagger.v3.oas.annotations.responses.ApiResponses;
import io.swagger.v3.oas.annotations.tags.Tag;
import org.springframework.http.MediaType;
import org.springframework.http.ProblemDetail;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.net.URI;

@RestController
@RequestMapping("/api/v1/products")
@Tag(name = "Products", description = "Gestión del catálogo de productos")
public class CreateProductEndpoint {

    private final Pipeline pipeline;

    public CreateProductEndpoint(Pipeline pipeline) {
        this.pipeline = pipeline;
    }

    @Operation(
        summary = "Crea un nuevo producto en el catálogo.",
        description = "Recibe los datos del producto, valida y persiste la entidad.")
    @ApiResponses({
        @ApiResponse(responseCode = "201", description = "Producto creado exitosamente",
            content = @Content(schema = @Schema(implementation = CreateProduct.Response.class))),
        @ApiResponse(responseCode = "400", description = "Datos inválidos o incompletos",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "422", description = "Error de validación",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "409", description = "Conflicto (ej: producto duplicado)",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "500", description = "Error interno del servidor",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @PostMapping(
        consumes = MediaType.APPLICATION_JSON_VALUE,
        produces = MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<CreateProduct.Response> create(
            @RequestBody CreateProductRequest request) {
        var command = request.toCommand();
        var response = pipeline.send(command);
        URI location = URI.create("/api/v1/products/" + response.id());
        return ResponseEntity.created(location).body(response);
    }
}
```

### Documentar DTOs con `@Schema`

```java
import io.swagger.v3.oas.annotations.media.Schema;

@Schema(description = "Solicitud para crear un producto")
public record CreateProductRequest(

    @Schema(description = "Nombre del producto (3-100 caracteres)", example = "Laptop XPS 13")
    String name,

    @Schema(description = "Descripción detallada (mínimo 10 caracteres)",
            example = "Ultrabook premium con pantalla 4K")
    String description,

    @Schema(description = "Precio en USD (> 0)", example = "999.99")
    BigDecimal price,

    @Schema(description = "Cantidad en stock (>= 0)", example = "15")
    int stock
) { }
```

> Los comentarios `/// <summary>` de C# se sustituyen por la anotación `@Schema`
> (a nivel de tipo y de campo). springdoc también puede leer Javadoc activando el
> plugin `springdoc-openapi-maven-plugin` + `therapi-runtime-javadoc`, pero las
> anotaciones son lo más directo.

## Ejemplo completo: Todos los CRUD documentados

```java
package com.shopmicro.catalog.features.products;

import an.awesome.pipelinr.Pipeline;
import com.shopmicro.catalog.features.products.create.CreateProduct;
import com.shopmicro.catalog.features.products.getbyid.GetProductById;
import com.shopmicro.catalog.features.products.list.ListProducts;
import com.shopmicro.catalog.features.products.update.UpdateProduct;
import com.shopmicro.catalog.features.products.delete.DeleteProduct;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.media.Content;
import io.swagger.v3.oas.annotations.media.Schema;
import io.swagger.v3.oas.annotations.responses.ApiResponse;
import io.swagger.v3.oas.annotations.responses.ApiResponses;
import io.swagger.v3.oas.annotations.tags.Tag;
import org.springframework.http.MediaType;
import org.springframework.http.ProblemDetail;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.net.URI;
import java.util.UUID;

/**
 * Controlador para gestionar productos del catálogo.
 */
@RestController
@RequestMapping("/api/v1/products")
@Tag(name = "Products", description = "Gestión del catálogo de productos")
public class ProductsController {

    private final Pipeline pipeline;

    public ProductsController(Pipeline pipeline) {
        this.pipeline = pipeline;
    }

    @Operation(summary = "Obtiene la lista de productos con paginación.")
    @ApiResponses({
        @ApiResponse(responseCode = "200", description = "Lista de productos obtenida"),
        @ApiResponse(responseCode = "400", description = "Parámetros de paginación inválidos",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @GetMapping
    public ResponseEntity<ListProducts.Response> list(
            @Parameter(description = "Número de productos a saltar")
            @RequestParam(defaultValue = "0") int skip,
            @Parameter(description = "Número de productos a devolver (máx 100)")
            @RequestParam(defaultValue = "10") int take) {
        var response = pipeline.send(new ListProducts.Query(skip, take));
        return ResponseEntity.ok(response);
    }

    @Operation(summary = "Obtiene un producto específico por su ID.")
    @ApiResponses({
        @ApiResponse(responseCode = "200", description = "Producto encontrado"),
        @ApiResponse(responseCode = "404", description = "Producto no encontrado",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @GetMapping("/{id}")
    public ResponseEntity<GetProductById.Response> getById(@PathVariable UUID id) {
        var response = pipeline.send(new GetProductById.Query(id));
        return ResponseEntity.ok(response);
    }

    @Operation(summary = "Crea un nuevo producto en el catálogo.")
    @ApiResponses({
        @ApiResponse(responseCode = "201", description = "Producto creado exitosamente"),
        @ApiResponse(responseCode = "400", description = "Datos inválidos",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "422", description = "Errores de validación",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "409", description = "Producto duplicado",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @PostMapping
    public ResponseEntity<CreateProduct.Response> create(
            @RequestBody CreateProductRequest request) {
        var response = pipeline.send(request.toCommand());
        URI location = URI.create("/api/v1/products/" + response.id());
        return ResponseEntity.created(location).body(response);
    }

    @Operation(summary = "Actualiza completamente un producto (PUT).")
    @ApiResponses({
        @ApiResponse(responseCode = "204", description = "Producto actualizado"),
        @ApiResponse(responseCode = "404", description = "Producto no encontrado",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class))),
        @ApiResponse(responseCode = "422", description = "Errores de validación",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @PutMapping("/{id}")
    public ResponseEntity<Void> update(
            @PathVariable UUID id,
            @RequestBody UpdateProductRequest request) {
        pipeline.send(request.toCommand(id));
        return ResponseEntity.noContent().build();
    }

    @Operation(summary = "Actualiza parcialmente un producto (PATCH).")
    @ApiResponses({
        @ApiResponse(responseCode = "204", description = "Producto actualizado"),
        @ApiResponse(responseCode = "404", description = "Producto no encontrado",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @PatchMapping("/{id}")
    public ResponseEntity<Void> partialUpdate(
            @PathVariable UUID id,
            @RequestBody PatchProductRequest request) {
        pipeline.send(request.toCommand(id));
        return ResponseEntity.noContent().build();
    }

    @Operation(summary = "Elimina un producto del catálogo.")
    @ApiResponses({
        @ApiResponse(responseCode = "204", description = "Producto eliminado"),
        @ApiResponse(responseCode = "404", description = "Producto no encontrado",
            content = @Content(schema = @Schema(implementation = ProblemDetail.class)))
    })
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable UUID id) {
        pipeline.send(new DeleteProduct.Command(id));
        return ResponseEntity.noContent().build();
    }
}
```

DTOs asociados (con `record`):

## Filtros de Swagger personalizados

### Agrupar endpoints por tags

```java
@Tag(name = "Products")
@RestController
public class ProductsController { ... }
```

O por operación:

```java
@Operation(tags = {"Products"})
@GetMapping
public ResponseEntity<?> list(...) { ... }
```

### Marcar operaciones como deprecadas

```java
@Deprecated
@Operation(summary = "Usa /api/v2/products en su lugar", deprecated = true)
@GetMapping("/old-endpoint")
public ResponseEntity<?> oldEndpoint(...) { ... }
```

## Autenticación en Swagger

Ya configurada en el bean `OpenAPI` anterior con el `SecurityScheme` `bearer-jwt`. En
Swagger UI aparecerá un botón **"Authorize"** para pegar el token JWT. Para exigirlo
solo en endpoints concretos:

```java
import io.swagger.v3.oas.annotations.security.SecurityRequirement;

@SecurityRequirement(name = "bearer-jwt")
@PostMapping
public ResponseEntity<?> create(...) { ... }
```

## Generar clientes desde OpenAPI

Herramientas como **OpenAPI Generator** pueden generar clientes para Java, TypeScript,
Python, etc.

### Desde línea de comandos

```bash
# Generar cliente Java
openapi-generator-cli generate -i http://localhost:8080/v3/api-docs \
  -g java -o ./generated-client

# Generar SDK TypeScript
openapi-generator-cli generate -i http://localhost:8080/v3/api-docs \
  -g typescript-axios -o ./generated-sdk
```

> También existe el plugin Maven `openapi-generator-maven-plugin` para generar el
> cliente como parte del build.

## Puntos clave

- **OpenAPI:** Estándar abierto, machine-readable
- **springdoc-openapi:** Genera la spec y sirve Swagger UI automáticamente
- **@Operation / @Schema:** Documentación en el código (sustituyen a los comentarios XML)
- **@ApiResponse:** Declara qué tipos de respuesta devuelve cada endpoint
- **@Tag:** Organiza endpoints en Swagger UI
- **JWT Bearer:** `SecurityScheme` + botón "Authorize" en Swagger UI
- **Versionado:** `/api/v1/` en la URL del endpoint
- **Documentación viva:** Sincronizada automáticamente con el código
